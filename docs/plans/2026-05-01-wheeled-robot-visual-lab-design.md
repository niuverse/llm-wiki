# Wheeled Robot Visual Lab Design

## Goal

Build an embedded academic-style visualization tool inside the wiki to help study how wheel-level geometric constraints become chassis-level kinematic matrices for wheeled mobile robots.

## Scope

The first version covers three cases:

- Differential drive: two fixed conventional wheels, lateral no-slip constraint, and wheel-speed mapping.
- Omni wheel platform: wheels with passive lateral rollers, showing why lateral constraints are relaxed.
- Steerable wheel: steering angle changes the rolling direction and the matrix row dynamically.

The tool is not a physics simulator. It visualizes planar kinematics and constraints, not contact dynamics, tire deformation, motor dynamics, or closed-loop control.

## Academic Visual Style

The diagram should read like an interactive textbook figure:

- top-view planar geometry;
- body frame `{b}`, world frame `{s}`, wheel centers, wheel tangent directions, lateral constraint normals, and local velocity vectors;
- equations placed beside the diagram, with variables matching the labels in the drawing;
- low-saturation colors, thin strokes, grid/axis references, and no decorative visual effects;
- compact legends and figure captions that support note-taking.

## Interaction Model

Each embedded lab instance renders into a placeholder element in a Markdown page. A small vanilla TypeScript/JavaScript module scans the page after load and initializes every matching element.

Recommended controls:

- robot type selector: `differential`, `omni`, `steerable`;
- sliders for body twist components $v_x$, $v_y$, and $\omega$;
- steerable-wheel slider for steering angle $\beta$;
- optional geometry sliders for wheel radius $r$ and half-track $l$;
- reset button for canonical textbook values.

The visualization updates live:

- wheel positions stay fixed relative to the chassis;
- local velocity at each wheel is computed from $V_b = [v_x, v_y, \omega]^T$;
- rolling and lateral directions are redrawn;
- each displayed matrix row updates numerically and symbolically where practical.

## Mathematical Model

Use planar rigid-body velocity at wheel contact point:

$$
v_i =
\begin{bmatrix}
v_x \\
v_y
\end{bmatrix}
+ \omega
\begin{bmatrix}
-y_i \\
x_i
\end{bmatrix}
$$

where $(x_i, y_i)$ is wheel $i$ in the body frame. A wheel direction unit vector $t_i$ maps chassis twist to rolling speed:

$$
\dot\phi_i = \frac{1}{r} t_i^T v_i
$$

A conventional fixed wheel also contributes a lateral no-slip constraint with lateral unit vector $n_i$:

$$
n_i^T v_i = 0
$$

For steerable wheels, $t_i$ and $n_i$ depend on steering angle $\beta_i$. For omni wheels, the lateral no-slip constraint is relaxed by passive roller motion, so the interface emphasizes the difference between a driven rolling row and a forbidden lateral row.

## Embedding Architecture

Use the existing Quartz site:

- Markdown page: `wiki/syntheses/wheeled-robot-visual-lab.md` with explanatory prose, equations, source links, and placeholder divs.
- Static script: `quartz/static/wheeled-robot-visual-lab.js`.
- Static styles: `quartz/static/wheeled-robot-visual-lab.css`.
- Quartz component hook: add a small global loader so the script and CSS are available on wiki pages.

The preferred implementation uses SVG generated by vanilla DOM APIs. This avoids adding dependencies and keeps the diagrams inspectable and printable.
